### Linux, Ansible, Terraform & Kubernetes Interview Questions

### Linux

1.  I want to know the kernel version of the Linux server. How would you
    check it?
2.  If you need to analyze the server performance and system load, what
    approach would you follow? Which commands do you usually use?
3.  When you run the `top` command, what kind of information can you get
    from it?
4.  Which command do you use to check memory usage?
5.  When you run the `free -m` command, what information does it
    provide?
6.  Which Linux distributions and versions have you worked on?
7.  Suppose you are trying to create a new file on the server, but you
    are unable to create either a file or a directory. There is
    sufficient disk space, and you are logged in as the root user. What
    could be the possible reasons?
8.  What is the purpose of the `mount -o remount,rw` command?


### Ansible

1.  What is the default path of the Ansible inventory file?
2.  In your current organization, how do you segregate inventory files
    for different environments?
3.  Can you explain how you would install the HTTPD service using an
    Ansible playbook?
4.  Which module or command do you use in an Ansible playbook to install
    the HTTPD package?


### Terraform

1.  How do you manage credentials securely in Terraform?
2.  Suppose a Terraform execution fails midway and the state file gets
    corrupted. How would you troubleshoot and recover it?


### Kubernetes & AWS Troubleshooting

1.  A deployment has completed successfully, but you are unable to
    access the application using its URL. How would you troubleshoot the
    issue from both the Kubernetes side and the AWS side?
2.  When you run the `kubectl describe pod` command, what kind of
    information does it provide?
3.  After checking the Pod, what would you verify next?
4.  What additional checks would you perform on the server or inside the
    Pod?
5.  Where do you check the application logs? What is the log location or
    path?
