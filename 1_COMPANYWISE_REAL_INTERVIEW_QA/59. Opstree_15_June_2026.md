### **Jenkins**

1. **Q: What are the different types of Groovy pipelines?**
* **A:** Declarative and Scripted pipelines.


2. **Q: How can you trigger a pipeline?**
* **A:** Using GitHub Webhooks, Poll SCM (polling configurations), and Cron tab method for time-based scheduling (e.g., at 6:00 PM every day).


3. **Q: What are your rollback strategies in Jenkins?**
* **A:** For Kubernetes, use `kubectl rollout undo` commands and verify with `kubectl rollout status`. For artifacts, store versioned artifacts in Nexus or ECR and have the pipeline redeploy the last known stable version. These commands are typically placed in the `post` deployment block of the Jenkinsfile.


4. **Q: Scenario: Run SonarQube and Veracode scanning simultaneously.**
* **A:** Use the `parallel` block in the Jenkinsfile to execute the stages at the same time instead of sequentially.


5. **Q: Scenario: Skip SonarQube scanning for pull requests from feature branches to main.**
* **A:** Use the `when` expression block with a condition inside the stage to evaluate the branch and return a boolean (`true`/`false`) to determine whether to skip the stage.



### **Linux**

6. **Q: Difference between `adduser` and `useradd`?**
* **A:** `useradd` is a low-level utility that modifies system files directly (like `/etc/passwd` and `/etc/shadow`) and is often used in scripts. `adduser` is a high-level, user-friendly wrapper that automatically creates home directories and sets default configurations.


7. **Q: How to check disk utilization and directory size?**
* **A:** `df -h` for overall disk usage and `du -sh <path>` for specific directory sizes.


8. **Q: Troubleshoot: Alert shows >90% usage, but `df -h` shows <80%.**
* **A:** Check for hidden or deleted files still held by processes using `lsof`, check mount points using `df -hT`, or inspect inode usage using `df -i`.



### **Kubernetes**

9. **Q: How does DNS resolution work in a cluster?**
* **A:** Handled by the `CoreDNS` service (running in `kube-system`). Pods use the internal DNS server to look up service names. When a service is queried, CoreDNS returns the cluster IP.


10. **Q: How to isolate applications?**
* **A:** Use Namespaces for logical partitioning. Use Network Policies to explicitly allow or deny traffic between pods/namespaces.


11. **Q: How to upgrade a cluster (specifically EKS)?**
* **A:** Plan the upgrade path, take an `etcd` backup (for self-managed clusters), and use tools like `eksctl` or AWS management consoles for managed control plane upgrades.


12. **Q: Liveness probes?**
* **A:** They check if the application is running and healthy. If the probe fails, Kubernetes restarts the pod.



### **AWS**

13. **Q: How does Auto Scaling Group (ASG) know when to scale?**
* **A:** Triggered by CloudWatch Alarms based on metrics (CPU/Memory). CPU metrics are default, while Memory requires a CloudWatch agent to push custom metrics.


14. **Q: Troubleshooting a lost PEM file for EC2 access?**
* **A:** If Session Manager (SSM) is configured, use it to log in without a key. Otherwise, stop the instance, detach the root volume, attach it to another instance, and perform manual recovery/key replacement.



### **Terraform**

15. **Q: Updating tags on an EC2 instance?**
* **A:** Terraform performs an "in-place" update via API calls to AWS without destroying or recreating the resource.


16. **Q: Preventing downtime during updates?**
* **A:** Use the `lifecycle` block with `create_before_destroy = true`.


17. **Q: Importing manual resources?**
* **A:** Use `terraform import`.
